## ファイル命名規則

### 基本形式
```
[連番]_[画面名]_[状態].png
```

### 例
```
01_取引一覧_初期表示.png
02_取引一覧_検索後.png
03_取引詳細_モーダル.png
04_取引詳細_返金確認.png
```

### 状態の例
- `初期表示` - 画面を開いた直後
- `検索後` - 検索・フィルタ適用後
- `モーダル` - モーダル/ダイアログ表示中
- `入力中` - フォーム入力中
- `エラー` - エラー表示
- `確認` - 確認ダイアログ
- `完了` - 処理完了画面

## 機密情報のマスキング

### 必須マスキング項目
- [ ] 実際の顧客名・会社名
- [ ] メールアドレス
- [ ] 電話番号
- [ ] カード番号（下4桁以外）
- [ ] 銀行口座情報
- [ ] APIキー・シークレット
- [ ] パスワード

### マスキング方法
1. **スクショ前**: テストデータに切り替え（推奨）
2. **スクショ後**: 画像編集でモザイク/塗りつぶし

## 収集時のメモ

各フォルダに `_memo.txt` を作成し、以下を記録すると分析精度が上がります：

```
画面名: 取引一覧
URL: /admin/transactions
主な用途: 全取引の検索・閲覧・CSV出力
よく使う機能: 日付範囲検索、ステータスフィルタ
あまり使わない機能: カラム並び替え
不満点・改善希望: 検索が遅い、CSVの項目が足りない
```
