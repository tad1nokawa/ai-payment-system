# AQUAGATES 分析チェックリスト

> スクリーンショット分析時の確認項目

---

## 画面ごとの分析項目

### 1. 基本情報
- [ ] 画面名/タイトル
- [ ] URL/パス
- [ ] 対象ユーザー（管理者/加盟店/代理店/一般）

### 2. レイアウト分析
- [ ] ヘッダー構成
- [ ] サイドバー/ナビゲーション
- [ ] メインコンテンツエリア
- [ ] フッター

### 3. 機能分析
- [ ] 表示機能（一覧/詳細/グラフ等）
- [ ] 検索・フィルタ機能
- [ ] CRUD操作（作成/読取/更新/削除）
- [ ] 一括操作
- [ ] 出力機能（CSV/PDF等）

### 4. UI要素
- [ ] テーブル（カラム構成）
- [ ] フォーム（入力項目）
- [ ] ボタン（アクション種別）
- [ ] モーダル/ダイアログ
- [ ] タブ/アコーディオン
- [ ] ステータスバッジ
- [ ] アラート/通知

### 5. データ項目
- [ ] 表示項目一覧
- [ ] 必須/任意の区別
- [ ] 項目の型（文字列/数値/日付等）
- [ ] 選択肢（プルダウン等）

### 6. 業務フロー
- [ ] 画面遷移（どこから来てどこへ行くか）
- [ ] 前提条件
- [ ] 後続処理

### 7. 権限
- [ ] 閲覧可能なロール
- [ ] 操作可能なロール
- [ ] 項目レベルの権限差異

---

## 新WFとの比較項目

### 対応状況
- [ ] 完全対応
- [ ] 部分対応（差分あり）
- [ ] 未対応（新規追加必要）
- [ ] 不要（削除検討）

### 差分内容
- [ ] レイアウト差異
- [ ] 機能差異
- [ ] 項目差異
- [ ] UI差異

### 対応方針
- [ ] 新WFに合わせる
- [ ] AQUAGATESに合わせる
- [ ] 新規設計

## 分析フロー

```
1. スクショ確認
   ↓
2. 基本情報記録
   ↓
3. 機能・項目抽出
   ↓
4. 新WFと比較
   ↓
5. 差分記録
   ↓
6. 優先度判定
   ↓
7. 対応方針決定
```

---

## よくある差分パターン

### パターン1: 項目の過不足
- AQUAGATESにあるが新WFにない項目
- 新WFにあるがAQUAGATESにない項目

### パターン2: 機能の過不足
- 一括操作の有無
- 検索条件の違い
- 出力形式の違い

### パターン3: UI/UXの違い
- 操作ステップ数
- 画面遷移の違い
- 表示形式の違い

### パターン4: 業務フローの違い
- 承認フロー
- 通知タイミング
- 自動/手動の違い
