## 動画分析 + スクリーンショット分析で判明した重要な差異

### 1. デポジット管理
AQUAGATESの加盟店検索にはデポジット売上比率(%)とデポジット期間(ヶ月)のカラムがある。加盟店ごとにデポジット（チャージバック等に備えた留保金）を管理している。

→ **AIpayment M08 精算管理にデポジット管理機能の追加を検討**

### 2. 代理店フィーの詳細モデル
- 手数料報酬種別: 「仕切り（仕切り以上50.00%）」
- ブランドごとの仕切り率: VISA 2.90% / MC 2.90% / JCB 4.20% / AMEX 4.20% / Diners 4.20%
- 初期費・月額設定（固定料金）あり

→ **M15 代理店管理のフィータブに反映済みだが、仕切りモデルの詳細を確認要**

### 3. サイト登録の追加フィールド
- オーナーグループ
- カード共有グループ（不要と判断済み）
- 商品カテゴリ（大/中）
- お問い合わせ先情報の設定（決済画面に表示）

→ **マルチサイト対応時にサイト登録フォームに反映要**

### 4. 決済制限のインポート/エクスポート
AQUAGATESにはCSVインポート/エクスポート機能があり、大量の制限を一括管理可能。

→ **M07 不正検知にCSVインポート/エクスポート機能の追加を検討**

### 5. 決済制限候補（自動検知）
不正検知ルールに基づいて自動的に制限候補が生成され、運営が「採用/不採用」を判断するフロー。

→ **AIpaymentではAI不正検知の出力として、M02例外キューまたはM07不正検知に統合**

### 6. リカーリングの詳細仕様
- 商品名: アプリ内課金名（例: net.skgroupinc.paters.payingmember740）
- サイクル種別: 指定日数 / 月額
- 決済間隔: 30日毎、90日毎など
- リカーリングトークン: ハッシュ形式
- カード変更URL: /ttlement/recurringcard/[トークン]
- リトライ予定日の管理

### 7. 日別集計のフィルタ条件
AQUAGATESではURL決済（スイフトパス経由）の集計を分離するフィルタがある。

→ **M11レポートにURL決済フィルタを追加検討**

### 8. 管理画面操作ログ（監査証跡）【新発見】
スクリーンショット#53で発見。管理画面上の全操作をログとして記録・検索できる機能。
- 検索条件: スタッフ名/IPアドレス/URL/操作対象ページ/操作日時(From-To)/種別(all/mail)
- 結果カラム: ID/スタッフ/操作日時/IPアドレス/種別/対象ページ/URL
- PCI DSS準拠やセキュリティ監査に必要な機能

→ **M13システム設定に「操作ログ」タブの追加を検討。または独立画面として実装**

### 9. 決済検索の豊富なフィルタ【スクリーンショット補完】
スクリーンショット#02で詳細が判明。AQUAGATESの決済検索は非常に多くの検索条件を持つ:
- マネーロンダリングフラグによるフィルタ
- チケット番号による検索
- 課金方法フィルタ
- 決済備考検索
- テスト決済の分離表示

→ **M06取引モニタリングのフィルタ条件の充実化を検討**

### 10. エラーコード一覧の大量データ【スクリーンショット補完】
スクリーンショット#45で確認。エラーコード一覧は非常に縦長のリストで、数百件のエラーコードが定義されている。各コードにリンクIDとエラーメッセージが紐付いている。

→ **M13システム設定のエラーコード管理は、ページネーション付きのリスト表示が必要**
