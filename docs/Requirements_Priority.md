# 要件優先度マトリクス v2.0

**作成日: 2026-02-16 / 最終更新: 2026-02-22**
**対象: AIpayment全機能の実装優先度**

> 全ドキュメント（System_Architecture / Screen_Specification / DB_Design / Core_System_Integration / AQUAGATES_Gap_Analysis / Screen_Logic_Gaps / Feature_List）を横断分析し、PSP事業のビジネスクリティカリティに基づき優先度を決定。

---

## 優先度定義

| 優先度 | 定義 | 目安 | 目標時期 |
|--------|------|------|---------|
| **P0: 必須** | ローンチに必ず必要 | 毎日使用 / なければ業務停止 | Phase 1（Week 1-12） |
| **P1: 重要** | 初期リリースに含めたい | 週次使用 / 代替手段なし | Phase 1後半（Week 8-16） |
| **P2: 標準** | Phase 2で対応可 | 月次使用 / 手動代替可 | Phase 2（Month 4-6） |
| **P3: 低** | 後回しで問題なし | ほぼ未使用 / nice-to-have | Phase 3（Month 7+） |
