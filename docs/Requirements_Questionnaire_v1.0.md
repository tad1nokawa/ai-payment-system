## 回答サマリー（2026-02-22時点）

### 確定した主要決定事項
1. **手数料**: 取引時点の料率適用、手数料率(%) + TR手数料(固定円/件) の2層構造
2. **精算**: 接続先依存の締め日、毎日AM2:00バッチ実行、翌営業日スライド、エラー分のみ例外キュー
3. **入金計算**: `決済高 − 手数料 − TR手数料 − CB差引 − リザーブ留保 + リザーブ解放` で確定
4. **セキュリティ**: PCI DSS v4.0準拠（セッション15分）、MFAはマスター管理のみ、IP制限はマスター+加盟店、パスワード90日更新、6回失敗→30分ロック
5. **不正検知**: 承認フロー維持（申請→承認→反映）
6. **CB閾値**: CB金額 > リザーブ残高 で例外キューにエスカレーション
7. **決済リンク**: 制限なし（有効/無効切替のみ）、決済完了時メール通知
8. **代理店**: 取引額ベースの報酬、月1回支払、運営が紐づけ変更可能
9. **運用**: 接続先切替時は新規取引一時停止、CSV上限10,000件、テストデータはtest_modeフラグ
10. **UI**: PC限定、i18n設計だけ準備、30秒ポーリング
11. **インフラ**: AWS PrivateLink、外部API→将来自社AI、RDS Multi-AZ、日次バックアップ+PITR 35日
